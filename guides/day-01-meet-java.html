01

What is "code," really?

Making Maggi noodles

  • 1 Switch on the gas stove
  • 2 Fill a vessel with 500ml water
  • 3 Wait until it boils
  • 4 Put the noodles inside
  • 5 Add the masala packet
  • 6 Wait 2 minutes, then turn it off

The same idea, as code

// a set of instructions, in order
startStove();
fillVessel(500);
waitUntilBoiling();
addNoodles();
addMasala();
wait(2);
stopStove();
Key idea: A program is just an ordered set of instructions to do one specific task. A programming language (Java, Python, C) is how humans write those instructions so a machine can follow them. Machines only understand binary (0s and 1s) — electrical signals that are either on or off — so every language you write eventually gets translated down to that.
02

Java at a glance

Created by

James Gosling, Sun Microsystems (now Oracle), 1995. Originally called Oak, project name "Green Team."

Platform independent

Compiled once into bytecode, that bytecode runs on any OS with a JVM — Write Once, Run Anywhere (WORA).

Object-oriented

Everything is modeled as objects — built on Class, Object, Inheritance, Polymorphism, Abstraction, Encapsulation.

Secure

Code runs inside the JVM sandbox — if something goes wrong, only the JVM is affected, never your operating system.

Robust

Handles memory automatically through garbage collection — you rarely manage memory by hand.

Multithreaded

Can run multiple tasks at the same time within one program — covered later in Collections & Multithreading.

Java is used across three flavors: J2SE (core Java — what you're learning now), J2EE (enterprise/full-stack web apps), and J2ME (micro/embedded devices).

03

JDK vs JRE vs JVM

JDK — JAVA DEVELOPMENT KIT
Everything you need to write and build Java
Includes the compiler (javac), debugger, and other dev tools — plus a full JRE inside it. This is what you install to start coding.
JRE — JAVA RUNTIME ENVIRONMENT
Everything you need to run Java
Core libraries + the JVM. If someone only wants to run a Java app (not write one), the JRE alone is enough.
JVM — JAVA VIRTUAL MACHINE
The engine that actually executes your bytecode
Reads the .class file line by line and translates it into instructions your specific OS understands. This is the piece that makes WORA possible.
Think of it like: JDK = a fully equipped kitchen (you can cook AND eat). JRE = a plate of food ready to eat (you can eat, not cook). JVM = your mouth and stomach — the part that actually "runs" the food, no matter what kitchen it came from.
04

Your first Java program

public class Demo { public static void main(String args[]) { System.out.println("This is my first program"); } }
public class Demo

Class declaration. "public" = access modifier (freely accessible). "class" is a fixed keyword. "Demo" is the class name — it must match the file name (Demo.java) and by convention starts with a capital letter.

public static void main

The main method — every Java program starts execution here. "static" means it runs without needing to create an object first. "void" means it returns nothing.

String args[]

Command-line arguments — a way to pass input into your program when it starts. You'll rarely touch this as a beginner, but it must always be present in main.

System.out.println()

System is a built-in class, out is an object inside it, println() is a method on that object. Whatever is inside the double quotes gets printed exactly as written.

println vs print: println() moves to a new line after printing. print() keeps the cursor on the same line, so the next output continues right after it.
05

From source code to a running app

1
Pick an editor

Notepad, Notepad++, or an IDE like Eclipse / IntelliJ. IDEs automate ~75% of the setup work.

2
Write the logic

Type your class, main method, and statements — this is your source code.

3
Save the file

Always with a .java extension, matching the public class name exactly.

4
Compile it

Converts source into bytecode. Catches syntax errors — brackets, semicolons, spelling, case sensitivity.

5
Execute it

The JVM runs the bytecode line by line, catching logical errors along the way.

// Step 1 — compile the source file into bytecode $ javac Hello.java → generates Hello.class (bytecode) // Step 2 — run the compiled bytecode $ java Hello → This is my first program

What is JIT? The Just-In-Time compiler lives inside the JVM. Instead of interpreting the same bytecode line over and over, it compiles frequently-used parts directly into native machine code — making your program run noticeably faster the longer it executes.
06

Try it yourself

  1. Write a program that prints your name, age, and city — each on a new line using println().
  2. Write a program using only print() statements and predict the output before running it — notice how everything joins onto one line.
  3. Change the class name and file name so they don't match, and observe the compiler error. This confirms the rule: file name = public class name.
  4. Add a single-line comment (//) above your printline explaining what it does — comments are ignored by the compiler.
Common beginner mistakes: forgetting the semicolon ;, mismatched { }, saving the file with a different name than the public class, or typing Main/MAIN instead of exactly main — Java is case-sensitive everywhere.

Key terms — flip to revise

Tap a card to flip it. Cover the term, say the definition out loud, then flip to check yourself.

Bytecode
tap to flip
The platform-independent output of javac (a .class file). Not machine code — the JVM reads it and translates it for whatever OS it's on.
JVM
tap to flip
Java Virtual Machine. Reads bytecode line by line and executes it on the actual OS underneath — the piece that makes WORA possible.
JIT
tap to flip
Just-In-Time compiler, inside the JVM. Compiles hot/frequently-used bytecode into native machine code so the program speeds up as it runs.
WORA
tap to flip
Write Once, Run Anywhere. You compile your source once into bytecode; any OS with a JVM installed can run that same bytecode unchanged.
javac
tap to flip
The Java compiler. Takes your .java source file, checks syntax, and produces a .class bytecode file with the same name.
Garbage Collection
tap to flip
Java's automatic memory manager. It reclaims memory used by objects no longer referenced, so you rarely allocate or free memory by hand.
