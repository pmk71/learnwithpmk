What is an Array?

Core Concept
Definition: An Array is an ordered, index-based collection of values (elements). In JavaScript, arrays are zero-indexed — the first element sits at index 0. Arrays are mutable (can change after creation), dynamic (grow/shrink automatically), and can hold mixed types in a single array since JS arrays are not strictly typed.

Under the hood, a JS array is actually a specialized object whose keys are stringified indices ("0", "1", ...) plus a special length property that auto-updates.

Array anatomy — index 0, 4, length 5
// ['click','fill','wait','assert','done']
// index:  0        1       2       3        4
// negative:-5      -4      -3      -2       -1
typeof & Array checks
typeof []               // 'object'
Array.isArray([])       // true — use this
[].length               // 0
Mixed types & length
[1,2,3].length          // 3
[1, 'hello', true, null, {name:'qa'}]
// arrays can hold ANY type together
Why it matters for automation: Playwright's locator.all(), page.$$(), and API responses (response.json()) almost always return arrays — mastering array methods is core to writing clean assertions and data-driven tests.

Array Creation

5+ Ways
Definition: Arrays can be created via a literal, the Array constructor, or by converting/generating from another iterable or array-like object.
1 · Array literal — most common
const steps = ['login', 'checkout', 'pay'];
2 · Array constructor — supports negatives
const a = new Array(3)     // [empty x 3]
Array(1,2,3)             // [1, 2, 3]
3 · Array.from() — from array-like / iterable
Array.from('abc')        // ['a','b','c']
Array.from({length:3}, (_,i) => i+1)
// [1, 2, 3]
Array.from(document.querySelectorAll('li'))
// NodeList → real array
4 · Array.of() — always creates elements (unlike Array())
Array.of(1,2,3)          // [1, 2, 3]
Array.of(7)              // [7]  (not empty x7!)
new Array(7)             // [empty x 7] — different!
5 · Spread from another array / set / string
const copy = [...tests];    // shallow copy
[...new Set([1,2,2,3])]  // [1,2,3] dedupe
6 · fill() to seed values
new Array(3).fill(0)    // [0, 0, 0]
Watch out: Array(7) creates a sparse array of length 7 with no actual elements, so .map() silently skips it. Use Array.from({length:7}) or Array(7).fill(undefined) when you need to iterate.

Access & Index

Read Elements
Bracket notation
tests[0]     // 'login'
tests[1]     // 'search'
tests[99]    // undefined (no error)
.at() — supports negative indices (ES2022)
tests.at(0)     // 'login'
tests.at(-1)    // 'checkout' (last)
tests.at(-2)    // 'search'
length
tests.length                    // 3
tests[tests.length - 1]         // 'checkout' (last)
Destructuring access
const [first, second, ...rest] = tests;
[first?.split(' ')[0] ?? 'checkout']
Swap via destructuring
[tests[0], tests[1]] = [tests[1], tests[0]];
Definition — array-like: An object with numeric keys and a length property (e.g. arguments, a NodeList) that isn't a "true" array and lacks array methods until converted with Array.from() or spread.

Spread & Rest

Operator
Definition: The ... syntax is spread when it expands an iterable into individual elements, and rest when it collects remaining elements into a new array. Same symbol, opposite direction.
Spread — expand arrays
const a = [1,2], b = [3,4];
[...a, ...b]     // [1, 2, 3, 4]
Clone (shallow copy)
const copy = [...a];
Add items around existing ones
[0, ...a, 3]      // [0, 1, 2, 3]
Spread into function args
Math.max(...[3,10,1])   // 10
Rest — collect remaining args
const [head, ...tail] = [1,2,3,4];
head // 1, tail // [2,3,4]

function logAll(...args){ /* args is real array */ }
Merge test suites / remove duplicates
const smoke = ['login','home'];
const full = [...smoke, 'cart', 'pay'];
[...new Set([1,2,2,3])]   // [1, 2, 3]

Mutating Methods — Change the Original Array

⚠ Mutates
Definition: A mutating method changes the array it's called on in place and often returns something other than the new array (e.g. the removed item, or the new length) rather than the array itself.
push() / pop()
const steps = ['login','cart'];
steps.push('checkout')
// 3 (new length)
steps.pop()
// 'checkout' (removed item)
shift() / unshift()
const q = ['B','C'];
q.unshift('A')   // 3 (new length)
q.shift()        // 'A' (removed item)
splice(start, deleteCount, ...items)
const arr = ['A','B','C','D'];
arr.splice(1,1)          // ['B'] removed
arr.splice(1,0,'X')     // insert at 1
arr.splice(2,1,'Y')     // replace at 2
sort() / reverse() / fill()
const nums = [3,1,4,1,5];
nums.sort()             // lexical by default!
nums.sort((a,b)=>a-b)   // numeric ascending
nums.reverse()          // reverses in place
new Array(3).fill(0)    // [0,0,0]
copyWithin() — rarely used
[1,2,3,4,5].copyWithin(0,3)
// [4,5,3,4,5] — copies slice within itself
Queue pattern (FIFO)
queue.push('test4')  // enqueue
queue.shift()          // dequeue
Watch out: sort() converts elements to strings by default, so [10, 2, 1].sort() gives [1, 10, 2], not numeric order. Always pass a comparator for numbers.

Immutable Variants (ES2023)

✓ Pure
Definition: ES2023 added non-mutating "copying" counterparts for the classic mutating methods. Each returns a new array and leaves the original untouched — safer for test data you reuse across specs.
toSorted() / toReversed()
const original = [3,1,2];
const sorted = original.toSorted((a,b)=>a-b);
// sorted: [1,2,3], original still [3,1,2]
toSpliced()
const tests = ['a','b','c'];
tests.toSpliced(1,1,'x')
// ['a','x','c'] — original untouched
with(index, value) — copy + replace one item
['login','cart','pay'].with(1,'checkout')
// ['login','checkout','pay']
Why it matters: In Playwright fixtures/data files, prefer toSorted()/with() over sort()/index assignment so shared test-data arrays aren't accidentally mutated by one spec and leaked into another.
Browser/Node support: Chrome 110+, Node 20+. Check your CI runner's Node version before relying on these in a framework.

Freezing & Immutability

Defensive Coding
Definition: Object.freeze() makes an array's existing elements read-only at the top level — any mutating method call fails silently (or throws in strict mode) rather than changing the data. Useful for protecting shared fixtures from accidental mutation across test files.
Object.freeze() on arrays
const ROLES = Object.freeze(['admin','user','guest']);
ROLES.push('super')     // fails silently (non-strict)
'use strict';
ROLES.push('super')     // TypeError: Cannot add property

Object.isFrozen(ROLES)   // true
Shallow freeze — nested objects stay mutable
const data = Object.freeze([{name:'Alice'}]);
data[0].name = 'Bob';    // ✗ works! nested object not frozen
data[0]           // {name: 'Bob'}

// Deep-freeze helper for full protection
function deepFreeze(arr){
  arr.forEach(item => typeof item === 'object'
    && item !== null && Object.freeze(item));
  return Object.freeze(arr);
}
Test automation use case: Freeze constant test-data arrays (role lists, environment configs, expected-value sets) at module load so an accidental .push()/.sort() in one spec can never leak state into another test file.

.map() — Transform Every Element

✓ Pure
Returns a new array of the same length with each element transformed by the callback. Does NOT mutate original. Most used array method in automation for data transformation.
Syntax
array.map((element, index, array) => {...})

const nums = [1, 2, 3];
nums.map(n => n * 2)      // [2, 4, 6]
nums.map(n => n.toString())
// ['1','2','3']

// with index
nums.map((n, i) => `${i}:${n}`)
// ['0:1','1:2','2:3']

// map objects
const users = [{name:'Alice'},{name:'Bob'}];
users.map(u => u.name)   // ['Alice','Bob']

// chain map → filter
[1,2,3,4].map(n => n ** 2).filter(n => n > 4)
// [9, 16]
Playwright
// Get text of all table rows
const rows = await page
  .locator('tbody tr')
  .allInnerTexts();

// Map to extract column values
const names = rows.map(r =>
  r.split('\t')[0]);

// Map prices to numbers
const prices = await page
  .locator('.price')
  .allInnerTexts();
const nums = prices.map(p =>
  parseFloat(p.replace('$','')));
// [29.99, 9.99, 49.0]

// Build test data array
const emails = ['a','b'].map(u =>
  `${u}@qa.com`);
// ['a@qa.com','b@qa.com']

.filter() — Keep Matching Elements

✓ Pure
Returns a new array with only elements where the callback returns truthy. Original is unchanged. Use to narrow down test data, filter passed/failed results, or find specific items.
Syntax
const scores = [45,82,60,91,38];
scores.filter(s => s >= 60)
// [82, 60, 91]
scores.filter(s => s < 60)
// [45, 38]

// filter objects
const tests = [
  {name:'login',   status:'pass'},
  {name:'search',  status:'fail'},
  {name:'checkout',status:'pass'},
];
tests.filter(t => t.status === 'fail')
// [{name:'search',status:'fail'}]

// filter strings containing substring
['login','cart','logout']
  .filter(s => s.includes('log'))
// ['login','logout']
Playwright
// Filter failed tests from results
const results = [
  {test:'login',   pass:true},
  {test:'register',pass:false},
  {test:'search',  pass:true},
];
results.filter(r => !r.pass)
// [{test:'register',pass:false}]

// Filter visible elements only
const allTexts = await page
  .locator('.item')
  .allInnerTexts();
const nonEmpty = allTexts
  .filter(t => t.trim() !== '')

// Filter test cases by tag
const smokeTests = testList
  .filter(t => t.tags
    .includes('@smoke'))

Search Methods — find, findIndex, indexOf & more

✓ Pure
Definition: Search methods locate an element or its position without creating a full filtered copy. Choose based on whether you need the value, the index, or just a yes/no answer.
find() — first match or undefined
const users = [
  {id:1,name:'Alice',role:'admin'},
  {id:2,name:'Bob',  role:'user'},
];
users.find(u => u.role==='admin')
// {id:1,name:'Alice',role:'admin'}
users.find(u => u.id===99)
// undefined

// findIndex — index of first match
users.findIndex(u => u.id===2)   // 1
users.findIndex(u => u.id===99)  // -1 (not found)

// findLast / findLastIndex (ES2023)
users.findLast(u => u.role==='user')
users.findLastIndex(u => u.role==='admin')
indexOf() / includes()
const steps = ['login','cart','pay'];
steps.indexOf('cart')     // 1 — primitive values
steps.indexOf('missing')  // -1
steps.lastIndexOf('pay')  // 2

steps.includes('cart')     // true
steps.includes('nope')     // false

// indexOf/includes use ===
[{a:1}].includes({a:1})  // false! (diff ref)

// in automation: check step exists
expect(steps.includes('pay'))
  .toBe(true);
find returns the value, findIndex returns the position, includes returns a boolean, indexOf only works for primitive equality (not deep object comparison).

Boolean Checks — some() & every()

Returns Boolean
.some() → true if at least one element passes. .every() → true if all elements pass. Both stop early (short-circuit) — essential for fast QA validation checks.
some() — at least one
const results = ['pass','fail','pass'];
results.some(r => r==='fail')
// true (any fail?)

[1,2,3].some(n => n > 2)     // true
[1,2,3].some(n => n > 9)     // false

// QA: any error at all?
errors.some(e =>
  e.includes('critical'))
every() — all must pass
const results = ['pass','pass','pass'];
results.every(r => r==='pass')
// true (all passed?)

['pass','fail'].every(r=>r==='pass')
// false

// QA: all prices > 0?
prices.every(p => p > 0)     // true

// All required fields filled?
fields.every(f =>
  f.trim() !== '')  // true/false
Empty-array edge case: [].every(fn) is always true ("vacuous truth"); [].some(fn) is always false.

.reduce() — Accumulate to a Single Value

✓ Pure
Reduces an array to a single value (number, string, object, array) by running a callback with an accumulator. Most powerful array method — can replicate map & filter.
Syntax
array.reduce((acc, cur, idx, arr) => {...}, initial)

// Sum
[1,2,3,4].reduce((acc,n) => acc+n, 0)   // 10

// Max value
[3,1,4,1,5].reduce((max,n) =>
  n > max ? n : max, -Infinity)          // 5

// Count occurrences
['a','b','a','c','a'].reduce((acc,v)=>{
  acc[v] = (acc[v]||0)+1; return acc;
}, {})
// {a:3, b:1, c:1}

// Flatten nested arrays
[[1,2],[3,4],[5]].reduce((acc,cur)=>
  [...acc,...cur], [])
// [1,2,3,4,5]
Playwright QA
// Sum all cart prices
const prices = [29.99, 9.99, 49.0];
const total = prices.reduce(
  (sum,p) => sum+p, 0)
// 88.98

// Group results by status
results.reduce((acc,t) => {
  (acc[t.status] ??= []).push(t.name);
  return acc;
}, {})
// {pass:['login'], fail:['pay']}

// Count pass/fail test statuses
results.reduce((acc,r) => {
  r.pass ? acc.pass++ : acc.fail++;
  return acc;
}, {pass:0, fail:0})
// {pass: 8, fail: 2}
Watch out: Forgetting to return acc inside the callback (when using a block body {}) silently breaks the accumulation on the next iteration.

forEach · flat · flatMap · slice · concat · join

✓ Pure (except forEach)
forEach — runs side effects, NO return
['login','pay'].forEach((step,i) => {
  console.log(`Step ${i+1}: ${step}`)
})

// Step 1: login / Step 2: pay
// forEach returns undefined
// Can't chain like .map()
flat / flatMap — flatten nested arrays
[1,[2,3]].flat()          // [1,2,3]
[1,[2,[3]]].flat()        // [1,2,[3]]
[1,[2,[3]]].flat(Infinity) // [1,2,3] all levels

// flatMap = map + flat(1)
['a b','c'].flatMap(s =>
  s.split(' '))
// ['a','b','c']
slice(start,end) — pure, non-mutating
const arr = ['a','b','c','d','e'];
arr.slice(1,3)      // ['b','c']
arr.slice(-2)       // ['d','e']
arr.slice()         // shallow copy
concat() / join()
[1,2].concat([3,4])    // [1,2,3,4]
[1].concat(2,[3,4])    // [1,2,3,4]

['step1','step2'].join(' -> ')
// 'step1 -> step2'
['a','b','c'].join('')   // 'abc'
splice vs slice: splice mutates and removes/inserts in place; slice is pure and returns a new sub-array — easy to mix up in interviews.

2D / Multi-dimensional Arrays

Nested Structures
Definition: JavaScript has no native matrix type — a "2D array" is simply an array whose elements are themselves arrays. Common for grid data, table snapshots, and seat/board-style test fixtures.
Creating a grid safely
// ✗ WRONG — all rows share the same reference
const bad = new Array(3).fill([0,0,0]);
bad[0][0] = 9;   // changes ALL rows!

// ✓ CORRECT — independent rows
const grid = Array.from({length:3}, () =>
  new Array(3).fill(0));
grid[0][0] = 9;  // only row 0 changes
Reading a table snapshot into a 2D array
// Playwright: read an HTML table into rows/cols
const rows = await page.locator('table tr').all();
const grid = [];
for (const row of rows) {
  const cells = await row.locator('td').allInnerTexts();
  grid.push(cells);
}
// [['Alice','Admin'], ['Bob','User']]
Flattening & transposing
const grid = [[1,2],[3,4],[5,6]];
grid.flat()        // [1,2,3,4,5,6]

// Transpose rows ↔ columns
const transposed = grid[0].map((_, c) =>
  grid.map(row => row[c]));
// [[1,3,5],[2,4,6]]
Iterating a 2D array
for (const row of grid) {
  for (const cell of row) {
    console.log(cell);
  }
}
Watch out: new Array(n).fill([]) gives every row a reference to the same inner array — mutating one row mutates them all. Always build rows with Array.from({length:n}, () => [...]) so each row is independent.

Destructuring — Keys, Values, Entries

Unpack
Definition: Destructuring unpacks values from arrays (by position) or objects (by key) into individual variables in one statement.
Array destructuring
const [a, b, c] = [1, 2, 3];

// Skip elements with commas
const [first, , third] = [1,2,3];
// first=1 third=3

// Default values
const [x=10, y=20] = [5];
// x=5 y=20

// Rest into array
const [head, ...tail] = [1,2,3,4];

// Swap values
let p=1, q=2;
[p, q] = [q, p];
// p=2 q=1

// In function params
function first2([a, b]) {
  return [a, b];
}
Object key/values/entries unpack
const config = {
  env: 'staging', timeout: 30000, retries: 3
};

Object.keys(config)
// ['env','timeout','retries']
Object.values(config)
// ['staging',30000,3]
Object.entries(config)
// [['env','staging'],['timeout',30000],['retries',3]]

// Loop over object
for (const [key,val] of
  Object.entries(config))
  console.log(`${key}: ${val}`)

// Convert array back to object
Object.fromEntries([['a',1]])
// {a: 1}

Grouping — Object.groupBy() / Map.groupBy() (ES2024)

✓ Pure · New
Definition: Groups elements of an iterable according to a callback's return value, producing an object (or a Map) whose keys are group names and values are arrays of matching items — replacing the manual .reduce() grouping pattern.
Object.groupBy()
const tests = [
  {name:'login', status:'pass'},
  {name:'pay',   status:'fail'},
  {name:'search',status:'pass'},
];
Object.groupBy(tests, t => t.status)
// { pass: [login, search], fail: [pay] }
Map.groupBy() — keeps non-string keys
const groups = Map.groupBy(
  [1,2,3,4,5], n => n % 2 === 0 ? 'even' : 'odd'
);
groups.get('even')   // [2, 4]
Requires a modern runtime (Node 21+ / evergreen browsers). Prior to this, the .reduce() pattern shown earlier is the standard fallback.

Array vs Set vs Map — Choosing the Right Structure

Decision Guide
Array: ordered, indexed, allows duplicates. Set: ordered, unique values only, no index access. Map: ordered key→value pairs where keys can be any type (unlike plain objects, which coerce keys to strings).
NeedBest structureWhy
Ordered list of test stepsArrayPosition/order matters, duplicates allowed
Unique tags or IDs, fast dedupeSethas() is O(1); auto-dedupes on insert
Lookup table (id → user object)Mapget() is O(1) vs .find()'s O(n)
Non-string keys (objects, DOM nodes)MapObject keys always stringify; Map keeps the real type
Set — uniqueness & fast membership checks
const seenIds = new Set();
for (const row of apiRows) {
  if (seenIds.has(row.id)) continue; // skip dupes
  seenIds.add(row.id);
}
seenIds.size    // unique count
Map — O(1) lookups instead of .find()
const userById = new Map(
  users.map(u => [u.id, u])
);
userById.get(2)      // instant lookup
// vs. users.find(u => u.id === 2) — O(n) scan

Promises & Async Array Patterns

Playwright Essential
Definition: Promise.all(), Promise.allSettled(), and Promise.race() all take an array of promises and return a single promise — the standard way to run multiple async Playwright actions together.
Promise.all() — fails fast on first rejection
// Run several independent async steps in parallel
const [homeText, aboutText] = await Promise.all([
  page.locator('#home').innerText(),
  page.locator('#about').innerText(),
]);

// If ANY promise rejects, Promise.all rejects immediately
Promise.allSettled() — waits for all, no fail-fast
const results = await Promise.allSettled(
  urls.map(url => page.goto(url))
);
results.filter(r => r.status==='rejected')
// collects every failure without stopping early
Promise.race() — first to settle wins
const winner = await Promise.race([
  page.waitForSelector('.success'),
  page.waitForSelector('.error'),
]);
// useful for "whichever toast appears first" checks
Sequential vs parallel array processing
// Sequential — one at a time, in order
for (const user of users) {
  await createUser(user);
}

// Parallel — all at once, order not guaranteed
await Promise.all(users.map(createUser));
Tip: Use Promise.all() when every action must succeed for the test to be valid. Use Promise.allSettled() when you want a full report of which of many parallel checks passed or failed, without one failure hiding the rest.

Playwright — Arrays in Real Automation Tests

Real World Patterns
allTexts() / count() / assert on collections
// Get ALL matching elements' text
const items = await page
  .locator('ul li').allInnerTexts();
expect(items.length).toBe(3);

// Assert specific item exists
expect(items.includes('About'))
  .toBe(true);

// Assert order
expect(items[0]).toBe('Home');

// Filter then assert count
const active = items
  .filter(i => i !== 'Disabled');
expect(active.length).toBeGreaterThan(0);
Data-driven / parameterized tests
// Test multiple users
const users = [
  {email:'admin@t.com', role:'admin'},
  {email:'guest@t.com', role:'guest'},
];

for (const {email, role} of users) {
  test(`login as ${role}`, async ({page}) => {
    await loginPage.login(email, 'pw');
    await expect(page).toHaveURL(`/${role}`);
  });
}
Table / list validation
// Validate table column values
const statuses = await page
  .locator('td.status').allInnerTexts();

// All active?
expect(statuses.every(
  s => s === 'Active')).toBe(true);

// Any error?
expect(statuses.some(
  s => s === 'Error')).toBe(false);

// Count by status
const counts = statuses.reduce((acc,s)=>{
  acc[s] = (acc[s]||0)+1; return acc;
}, {});
// {Active:5, Pending:2}
API + array response validation
// Validate API response array
const res = await request.get('/users');
const body = await res.json();
expect(body.length).toBeGreaterThan(0);

// Check specific admin exists
const admin = body.find(
  u => u.role === 'admin');
expect(admin).toBeDefined();

// All fields required?
expect(body.every(u =>
  u.id && u.name && u.email)).toBe(true);

// Extract IDs
const ids = body.map(u => u.id);
// [1, 2, 3, ...]

Sorting — Safe vs Unsafe Patterns

⚠ Mutates Unless Noted
⚠ .sort() mutates original
const original = [3,1,2];
const sorted = original.sort();
// original is now ALSO mutated!

// Default: converts to strings, lex order
[10, 2, 1].sort()   // [1, 10, 2] WRONG

// Numeric ascending
nums.sort((a,b) => a-b)   // [1, 10, 2] -> fixed

// Numeric descending
nums.sort((a,b) => b-a)

// Sort strings
['banana','apple','cherry'].sort()

// Locale-aware compare
['é','a','z'].sort((a,b)=>
  a.localeCompare(b))

// Sort objects by property
users.sort((a,b) =>
  a.name.localeCompare(b.name))
✓ toSorted() — returns new, safe
const original = [3,1,2];

// sorted [1,2,3], original still [3,1,2]

// toReversed() — returns new

  [...original].sort((a,b)=>b-a);

// Safe pattern (pre-ES2023)

  [...original].sort((a,b)=>a-b);

Best practice: Never sort shared fixture/test data directly. Clone first with [...arr] (or use toSorted()) so parallel Playwright workers don't race on a mutated array.

Big-O Performance Guide

Complexity
Definition: Big-O notation describes how an operation's cost grows as array size n grows. Knowing which array methods are O(1), O(n), or O(n²) helps you avoid slow test suites when processing large datasets or API responses.
OperationComplexityNotes
arr[i], .at(i), .lengthO(1)Direct index access — constant time
.push() / .pop()O(1)Operates on the end of the array
.unshift() / .shift()O(n)Every remaining element must re-index
.splice() (middle)O(n)Shifts all elements after the split point
.map / .filter / .forEach / .reduceO(n)One pass through every element
.find / .indexOf / .includesO(n)Linear scan — stops early on match
.sort()O(n log n)V8 uses TimSort — stable since ES2019
Nested loops / nested .find()O(n²)Common accidental slowdown — replace inner loop with a Map/Set lookup
Set.has() / Map.get()O(1)Hash-based — why Set/Map beat Array for repeated lookups
Common trap: Doing arr1.filter(a => arr2.find(b => b.id === a.id)) is O(n·m) — for large datasets, convert arr2 to a Map or Set first so the lookup becomes O(1) per item, turning the whole operation into O(n).

Interview Q&A

Accordion
What is the difference between map() and forEach()?▾
map() returns a brand-new array built from the callback's return values and is chainable; forEach() always returns undefined and exists purely for side effects (like logging or pushing into an outside array).
Which array methods mutate the original array?▾
push, pop, shift, unshift, splice, sort, reverse, fill, copyWithin all mutate in place. Everything else (map, filter, slice, concat, reduce, the ES2023 to* methods, etc.) returns a new array or value and leaves the original untouched.
Why does [10, 2, 1].sort() give [1, 10, 2]?▾
Without a comparator, sort() converts every element to a string and compares them lexicographically (Unicode code point order), so "10" sorts before "2". Always pass (a,b) => a - b for numeric sorting.
How do you remove duplicates from an array?▾
The idiomatic one-liner is [...new Set(arr)], since a Set only stores unique values. For objects/arrays of objects, dedupe on a key with reduce() or a Map keyed by an identifying field.
What's the difference between slice() and splice()?▾
slice(start, end) is pure — it returns a shallow copy of a portion without touching the original. splice(start, deleteCount, ...items) mutates the array in place to remove and/or insert elements, and returns the removed elements.
Why does array.forEach(async fn) not wait for each iteration?▾
forEach ignores the promise returned by an async callback — it fires all callbacks immediately without waiting. For sequential async work, use a for...of loop; for parallel async work, use Promise.all(arr.map(asyncFn)).
How do you check if a value is truly an array?▾
Use Array.isArray(value) — never typeof, since typeof [] returns 'object', the same as for plain objects.
How would you flatten a deeply nested array?▾
arr.flat(Infinity) flattens all nesting levels in one call. For one level of nesting combined with a transform, use flatMap() instead of map().flat() for slightly better performance and clarity.
When would you use a Map or Set instead of an array?▾
Reach for a Set when you only care about unique membership (fast has() checks, auto-dedupe). Reach for a Map when you need repeated key-based lookups — map.get(id) is O(1) versus array.find()'s O(n) scan. Use an array when order and duplicates both matter.
What's the difference between Promise.all() and Promise.allSettled()?▾
Promise.all() rejects immediately if any promise in the array rejects, discarding the other results. Promise.allSettled() always waits for every promise and returns a status for each one, so you can see every success and every failure instead of just the first error.
In Playwright, how do you assert every element in a list meets a condition?▾
Collect the values with await locator.allInnerTexts() (or allTextContents()), then run .every() on the resulting array and assert the boolean, e.g. expect(statuses.every(s => s === 'Active')).toBe(true).
What's the safest way to sort a shared test-data array without side effects?▾
Clone before sorting: [...arr].sort(cmp), or use the ES2023 non-mutating arr.toSorted(cmp), so the original fixture isn't silently changed for other tests running in parallel.
