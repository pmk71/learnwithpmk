Setup Guide · Do this once

Install & configure Java — so every lesson just works

Before you write a line of code, your machine needs three things: a JDK installed, your PATH pointed at it, and a terminal that says yes when you ask it for a Java version. This walks through all three, for whichever OS you're on.

01

What you're actually setting upThree pieces, one goal: your terminal can compile and run Java anywhere

Install the JDK

The Java Development Kit — compiler, JVM, and libraries in one package. Nothing else works without this.

Set JAVA_HOME & PATH

Tells your OS where Java lives and lets you type java from any folder, not just the install directory.

Verify it works

Run two commands in a terminal. If both respond with a version number, you're fully set up.

02

Download & install the JDKPick your OS — the steps only apply to that one

1
Download the JDK

Go to Adoptium.net (or Oracle's JDK page) and download the Windows x64 Installer (.msi) for the latest LTS version.

2
Run the installer

Double-click the .msi file. Keep the default install path — usually C:\Program Files\Eclipse Adoptium\....

3
Tick "Set JAVA_HOME"

Most installers show a checkbox for this during setup — enable it and the installer handles Step 03 for you automatically.

4
Open a fresh terminal

Close any Command Prompt / PowerShell windows that were open before installing, then open a new one so it picks up the updated PATH.

1
Install via Homebrew (recommended)

Open Terminal and run brew install openjdk@21. No Homebrew? Install it first from brew.sh.

2
Symlink it (Homebrew asks you to)

Homebrew prints a sudo ln -sfn command after install — copy-paste that exact line to link the JDK system-wide.

3
Add JAVA_HOME to your shell profile

In ~/.zshrc, add export JAVA_HOME=$(/usr/libexec/java_home), save, then run source ~/.zshrc.

4
Restart Terminal

Quit and reopen Terminal (or open a new tab) so the profile changes take effect.

1
Install via package manager

Debian/Ubuntu: sudo apt install openjdk-21-jdk. Fedora: sudo dnf install java-21-openjdk-devel.

2
Find the install path

Run update-alternatives --list java (or readlink -f $(which java)) to see exactly where it landed.

3
Export JAVA_HOME in ~/.bashrc

Add export JAVA_HOME=/usr/lib/jvm/java-21-openjdk and export PATH=$JAVA_HOME/bin:$PATH, then source ~/.bashrc.

4
Reload your shell

Open a new terminal tab, or run source ~/.bashrc in the current one, to pick up the new PATH.

03

How JAVA_HOME & PATH connectTwo settings, one purpose: let any terminal find your JDK

JDK folder

If JAVA_HOME is missing, tools like Maven, Gradle, and IDEs can't find your JDK even if java works in a plain terminal — that's why both matter, not just one.

04

Verify the installTwo commands. Both should print a version number, not an error

// check the JDK compiler $ javac -version → javac 21.0.3 // check the JVM / runtime $ java -version → openjdk version "21.0.3" 2024-04-16 → OpenJDK Runtime Environment (build 21.0.3+9)

Predict what you'll see, then click ▶ Run to reveal it and check yourself.

Command not found / "java is not recognized"? +
This means PATH doesn't include your JDK's bin folder yet. Re-check Step 03 above, then open a brand new terminal window — old ones keep the stale PATH from before you made the change.
05

Common setup errorsClick to expand — these cover 90% of first-time install issues

"JAVA_HOME is not set" from Maven/Gradle/an IDE +
The java command can work from PATH alone, but build tools specifically read the JAVA_HOME variable. Set it explicitly (Step 03) even if java -version already works.
Multiple Java versions are conflicting +
Run java -version and compare it against what JAVA_HOME points to. On Windows, check Path order in Environment Variables — whichever bin folder appears first wins.
Changes aren't taking effect +
Environment variables load when a terminal starts — they don't apply retroactively. Always close and reopen your terminal (or run source on your shell profile) after editing them.
"javac" works but "java" doesn't (or vice versa) +
You likely installed a JRE-only package at some point, which has java but not javac. Reinstall the full JDK — it always includes both.
