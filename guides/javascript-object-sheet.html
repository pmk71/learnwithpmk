01

What Is an Object?

📘

Definition: An object is an unordered collection of key–value pairs called properties. Keys are strings (or Symbols); values can be any type, including functions, which are then called methods. Objects are mutable and are always passed and compared by reference, never by value.

Anatomy of an Object

const config = {
  env: 'staging',   // string
  timeout: 30000,   // number
  headless: true,    // boolean
  browsers: ['chrome'], // array
  login: async () => {},// method
  proxy: null       // null
};

Every value in the object is a property; if the value is a function, it's called a method.

The typeof null Gotcha

typeof {}          // → "object"
typeof null        // → "object" ⚠ bug!
{} === {}          // → false
typeof []          // → "object" (arrays too)

This is a long-standing JS bug preserved for backwards compatibility — always check value === null explicitly.

02

Object Creation

Five common ways to create objects, from the simplest literal syntax to factory functions.

Object Literal

const person = {
  name: 'John',
  age: 30,
  greet() { console.log('Hi'); }
};

The most common way to create an object — best for one-off, simple structures.

Constructor Function

function Person(name, age) {
  this.name = name;
  this.age = age;
}
const p = new Person('John', 30);

Uses the new keyword to stamp out multiple similar instances.

Object.create()

const proto = {
  greet() { /* ... */ }
};
const obj = Object.create(proto);

Creates a new object with an explicit prototype — no constructor call involved.

Class Syntax (ES6)

class Person {
  constructor(name) {
    this.name = name;
  }
}
const p = new Person('John');

Syntactic sugar over prototype-based inheritance — cleaner, more readable syntax.

Factory Function

function makeUser(name, role) {
  return { name, role, active: true };
}
makeUser('Alice', 'admin');
// → {name:"Alice", role:"admin", active:true}

A plain function that returns a new object — no new or this needed. Great for test fixtures.

03

Property Access & Manipulation

Reading, writing, deleting, and checking properties on an object.

Dot vs. Bracket Notation

obj.property
obj['property']
obj[variableKey]  // dynamic key

Bracket notation is required when the key is dynamic or not a valid identifier (e.g. "base-url").

Adding / Updating

obj.newProp = 'value';
obj['newProp'] = 'value';

Assigning to a non-existent key creates it; assigning to an existing key updates it.

Deleting Properties

delete obj.property;

Removes the property entirely (not just sets it to undefined).

Checking Existence

'prop' in obj
obj.hasOwnProperty('prop')
Object.hasOwn(obj, 'prop') // ES2022
obj.prop !== undefined

in checks the prototype chain too; hasOwnProperty / Object.hasOwn check only the object itself.

04

Optional Chaining & Nullish Coalescing

🛡️

Definition: Optional chaining (?.) stops evaluating and returns undefined the moment it hits null/undefined, instead of throwing. Nullish coalescing (??) provides a fallback only for null/undefined — not for other falsy values like 0 or "".

Optional Chaining (?.)

const city = user?.address?.city;
user?.getRole?.();     // safe method call
items?.[0];              // safe array access
// Without ?. — throws if user is null:
user.address.city      // ✘ TypeError!

Essential when handling API responses in tests where fields may be missing.

Nullish Coalescing (??)

null ?? 'default'       // → "default"
0 ?? 'default'          // → 0   ✓ (not falsy)
'' ?? 'default'         // → ''  ✓
0 || 'default'          // → "default" ⚠ replaces 0

Prefer ?? over || whenever 0, "", or false are valid values.

Nullish Assignment (??=)

let x = null;
x ??= 'set now'; // → "set now"

// Combine ?. with ??
cfg?.timeout ?? 5000;
user?.role?.toUpperCase() ?? 'GUEST';

Only assigns when the variable is currently null or undefined.

05

Destructuring & Spread

📦

Definition: Destructuring unpacks properties from an object into standalone variables in one line — supports renaming, default values, nested unpacking, and rest collection. Spread (...obj) does the reverse: it copies enumerable own properties out into a new object.

Basic Destructuring

const { name, age = 18 } = person;
const { name: userName } = person; // rename
const { name: n, ...rest } = person; // rest

Extracts properties into variables, with optional defaults, renaming, and a rest collector.

Nested & Function Params

const { data: { user: { name } } } = response;

function login({ email, password, retries = 3 }) {
  console.log(email, retries);
}
login({ email: 'a@b.com', password: 'pw' });

Destructuring directly in function parameters keeps signatures short and self-documenting.

Spread Operator

const clone = { ...base };
const merged = { ...base, ...extra };
const override = { ...base, env: 'prod' };

Later keys override earlier ones — creates a shallow copy (nested objects are still shared references).

Destructure in Loops

const users = [{name:'A', role:'admin'}];
for (const { name, role } of users) {
  console.log(name, role);
}

Common pattern for data-driven Playwright/Jest tests iterating over test-case objects.

07

Iteration

Different ways to loop through — or transform — an object's properties.

for...in

for (const key in obj) {
  console.log(key, obj[key]);
}

Iterates enumerable keys, including inherited ones — guard with hasOwnProperty.

Object.entries + forEach

Object.entries(obj).forEach(
  ([key, value]) => console.log(key, value)
);

The modern, safest way — only own enumerable properties, no chain leakage.

Transform via fromEntries

Object.fromEntries(
  Object.entries(cfg)
    .map(([k, v]) => [k, v * 2])
);
// double every numeric value

Map/filter the entries array, then rebuild the object — the immutable way to transform objects.

Count / Filter Properties

Object.values(scores).filter(v => v > 100).length;
Object.keys(scores).length; // property count

Handy for asserting response shape — e.g. "response has exactly 3 keys" in a test.

08

Getters, Setters & this

🔑

Definition: Getters and setters intercept property reads and writes so you can compute, validate, or log on access — they're used exactly like normal properties but run function logic underneath. this refers to the object a method is called on, decided at call time, not definition time.

Method Shorthand & this

const testConfig = {
  env: 'staging',
  getUrl() {
    return `https://${this.env}.app.com`;
  }
};
testConfig.getUrl(); // → "https://staging.app.com"

this = the object the method was called on.

Getter / Setter

const config = {
  _env: 'staging',
  get baseUrl() {
    return `https://${this._env}.app.com`;
  },
  set env(val) {
    if (!['staging','prod'].includes(val))
      throw new Error('Invalid env');
    this._env = val;
  }
};
config.env = 'prod';   // runs the setter
config.baseUrl;        // → "https://prod.app.com"

Accessed like plain properties — no () — but let you validate input or compute derived values.

⚠ Arrow Functions & this

const bad = {
  val: 42,
  get: () => this.val // ✘ undefined!
};

Arrow functions don't have their own this — they inherit it from the enclosing (outer) scope, not the object.

Async Methods

const testConfig = {
  async fetchData() {
    const url = this.getUrl();
    return fetch(url);
  }
};

Common in Page Object Models where async actions need access to this.selectors.

09

Immutability

🔒

Definition: By default, every object property can be changed, added, or removed. Object.freeze() and Object.seal() restrict that — both operate shallowly, meaning nested objects remain fully mutable unless frozen separately.

immutable

Object.freeze()

const obj = Object.freeze({ a: 1 });
obj.a = 2;   // silently fails (strict: throws)
Object.isFrozen(obj); // → true

Nothing can be changed, added, or removed. Great for exporting constants/enums.

restricted

Object.seal()

const obj = Object.seal({ a: 1 });
obj.a = 2;    // ✔ allowed — edits OK
obj.b = 3;    // ✘ blocked — no new keys
delete obj.a; // ✘ blocked — no deletion
Object.isSealed(obj); // → true

Existing properties can still be edited, but nothing can be added or removed.

gotcha

Shallow Freeze Trap

const cfg = Object.freeze({
  env: 'dev',
  nested: { port: 3000 }
});
cfg.nested.port = 9999; // ✔ works! not deep

To deep-freeze, recursively call Object.freeze() on every nested object.

10

Property Descriptors

⚙️

Definition: Every property secretly carries a descriptor object controlling its behavior: value, writable (can it change?), enumerable (does it show up in loops/Object.keys?), and configurable (can it be deleted or reconfigured?). Object.defineProperty() lets you set these directly.

Object.defineProperty()

Object.defineProperty(obj, 'version', {
  value: '1.0.0',
  writable: false,   // can't change
  enumerable: true,   // shows in keys()
  configurable: false // can't delete
});
obj.version = '2'; // silently fails

More granular than Object.freeze() — lock down a single property instead of the whole object.

Inspecting Descriptors

Object.getOwnPropertyDescriptor(obj, 'version');
// → { value:"1.0.0", writable:false, ... }

Object.getOwnPropertyDescriptors(obj);
// → descriptors for ALL own properties

Useful for debugging why a property "won't update" in a test fixture.

11

Prototypes & Inheritance

🧬

Definition: Every object has an internal link — its prototype — to another object it inherits properties and methods from. When you access a property JS doesn't find, it walks up this prototype chain until it finds it or reaches null.

Prototype Chain

function Animal(name) { this.name = name; }
Animal.prototype.speak = function() {
  console.log(this.name + ' makes a sound');
};
const dog = new Animal('Rex');
dog.speak(); // found via prototype chain

dog doesn't own speak — it's found one level up the chain.

class extends

class BasePage {
  constructor(page) { this.page = page; }
  async goto(url) { await this.page.goto(url); }
}
class LoginPage extends BasePage {
  async login() { /* uses this.page */ }
}

Class inheritance is prototype inheritance under the hood — common in Page Object base classes.

12

Symbols & Computed Property Names

🔣

Definition: A Symbol creates a guaranteed-unique key, useful for near-private properties that won't collide or show up in normal enumeration. Computed property names ([expr]: value) let you use a variable or expression as an object key at creation time.

Computed Property Names

const key = 'status';
const obj = { [key]: 'active' };
// → { status: "active" }

const env = 'staging';
const urls = {
  [env]: 'https://staging.app.com'
};

Common for building lookup objects keyed by a dynamic environment or ID.

Symbol Keys

const ID = Symbol('id');
const user = { [ID]: 42, name: 'Alice' };

user[ID];             // → 42
Object.keys(user);    // → ["name"] (no Symbol!)
Object.getOwnPropertySymbols(user); // → [Symbol(id)]

Symbols are excluded from Object.keys, for...in, and JSON.stringify — a soft-private key.

13

Proxy & Reflect

🪞

Definition: A Proxy wraps an object and lets you intercept fundamental operations — reading a property, setting one, checking in, deleting — via trap functions. Used for validation, logging, virtual properties, or reactive frameworks.

Basic Proxy

const handler = {
  get(target, prop) {
    if (prop in target) return target[prop];
    return `Key "${prop}" not found`;
  },
  set(target, prop, val) {
    if (typeof val !== 'string')
      throw new Error('String only!');
    target[prop] = val;
    return true;
  }
};
const cfg = new Proxy({ env: 'dev' }, handler);
cfg.env;      // → "dev"
cfg.missing;  // → 'Key "missing" not found'

Useful for building strict test-config objects that fail loudly on typos.

Reflect (companion API)

Reflect.get(obj, 'env');
Reflect.set(obj, 'env', 'prod');
Reflect.has(obj, 'env');      // like 'in'
Reflect.ownKeys(obj);          // strings + symbols

Reflect mirrors the same operations Proxy traps intercept — often used together inside trap handlers.

14

Cloning & Comparison

Copying objects safely and comparing them correctly.

Shallow vs Deep Clone

const shallow = { ...obj };
const deep = structuredClone(obj); // ✓ safe
// or (loses functions/undefined):
const d2 = JSON.parse(JSON.stringify(obj));

Shallow clones still share nested object references — use structuredClone for a true deep copy.

Reference Equality

{ a: 1 } === { a: 1 } // false — different refs
const x = { a: 1 };
const y = x;
x === y // true — same reference

Objects compare by reference, not value — use a deep-equal utility (or toEqual in tests) to compare contents.

Object.is() vs ===

Object.is(NaN, NaN);   // → true  (unlike ===)
NaN === NaN;            // → false
Object.is(0, -0);      // → false
0 === -0;               // → true

A rare edge case, but a favorite interview trick question.

Object vs Map

🗺️

Definition: Map is a dedicated key-value collection, distinct from plain objects — it accepts any value as a key (not just strings/Symbols), preserves insertion order reliably, and has a built-in .size.

FeatureObjectMap
Key typesString / Symbol onlyAny value (object, function, etc.)
SizeObject.keys(o).lengthmap.size
Iteration orderMostly insertion (edge cases with numeric keys)Guaranteed insertion order
Default keysInherits from Object.prototypeNo default keys — clean map
Iterationfor...in / Object.entriesDirectly iterable (for...of)
Best forStructured records, JSON dataFrequently changing key-value data

Common Pitfalls & Best Practices

⚠ Gotchas

// typeof null === "object" — bug!
typeof null // "object"

// Objects pass by reference
const a = { x: 1 };
const b = a;
b.x = 99;
a.x // → 99 ⚠ same ref!

// Spread is shallow only
const c = { ...a }; // nested objs still shared

// {} === {} is always false
{} === {} // → false

✓ Best Practices

// Use ?. for safe access
body?.user?.name ?? 'unknown';

// structuredClone for deep copy
structuredClone(obj);

// Object.hasOwn over hasOwnProperty
Object.hasOwn(obj, 'key'); // ES2022

// Object.freeze for constants
const ROUTES = Object.freeze({
  login: '/login', dashboard: '/dashboard'
});

// Destructure in fn params — self-documenting
async function login({ email, password }) {}

// JSON.stringify to compare by value
JSON.stringify(a) === JSON.stringify(b);

Interview Q&A

Click a question to reveal the answer.

Q. What's the difference between Object.freeze() and const?+

const only prevents reassigning the variable itself; the object it points to can still be mutated. Object.freeze() prevents the object's own properties from being changed, added, or removed.

Q. How do you check if a variable is an object vs an array?+

Use Array.isArray(value) — arrays are technically objects, so typeof returns 'object' for both.

Q. What does hasOwnProperty guard against?+

It excludes properties inherited from the prototype chain, so a for...in loop only processes the object's own data.

Q. Why use Object.entries() over for...in?+

It returns only own enumerable properties as an array you can safely map, filter, or reduce — no risk of inherited keys leaking in.

Q. How would you deep-compare two objects in a test assertion?+

Most test frameworks (Jest, Playwright's expect) provide a deep-equality matcher like toEqual() that recursively compares structure and values rather than references.

Q. When would you reach for a getter/setter instead of a plain property?+

When a value needs validation on write (a setter rejecting bad input) or needs to be computed from other properties on read (a getter deriving a full URL from a base + path), rather than stored directly.

Q. What's the real difference between structuredClone() and JSON.parse(JSON.stringify())?+

structuredClone() handles more types correctly — Dates, Maps, Sets, circular references — while the JSON trick silently drops functions and undefined values and throws on circular references.

Q. Why might you use a Proxy in a test framework?+

To fail fast on typos — e.g. a config Proxy that throws immediately when a test reads a key that doesn't exist, instead of silently returning undefined and failing later with a confusing error.

Q. Does spreading an object copy its methods?+

Yes, but only as references to the same function — spread copies own enumerable properties, including function values, but doesn't clone the functions themselves (functions aren't affected by shallow vs deep copy since they're not mutated).

Q. Why does for...in sometimes iterate more keys than Object.keys() returns?+

for...in walks the full prototype chain and includes inherited enumerable properties, while Object.keys() only returns the object's own enumerable keys.
